# 带货主播算税助手

一个面向直播带货收入的税负测算小工具，用来对比同一笔平台报送年营收在三种身份下的税费和到手金额：

- 有限公司
- 个体工商户
- 自然人主播

工具重点不是替代税务申报，而是帮助用户快速理解：在相同营收、成本、推广费和扣除条件下，不同身份的税负差异有多大。

## 在线试用

[打开带货主播算税助手](https://wfpet169.github.io/tax-burden-calculator/)

## 主要功能

- 对比三种身份的合计税费、税负率和测算到手金额。
- 广告/推广费同时展示“全额税前扣除”和“按年营收 15% 限额扣除”两种口径。
- 自然人主播按劳务报酬并入综合所得口径估算，不套用广告费 15% 限额。
- 支持查看每种身份的核查算式，方便逐项复核。
- 计算规则独立放在 JSON 文件中，便于后续按法规变化更新。
- 纯前端运行，不需要后端服务，输入数据不会上传。

## 快速开始

进入项目目录后启动静态服务：

```powershell
python -m http.server 8765
```

浏览器访问：

```text
http://127.0.0.1:8765/
```

也可以部署到 GitHub Pages、Nginx、Vercel 等任意静态站点服务。

## 使用说明

1. 填写经营数据：
   年营收按平台报送给税务机关的金额填写；其他成本/费用和广告/推广费分开填写。

2. 选择税务口径：
   根据实际情况选择增值税身份、城建税地区，以及是否适用六税两费、小型微利企业、个体工商户减半等优惠。

3. 填写个税扣除：
   录入基本减除费用、专项扣除、专项附加扣除和其他扣除，用于个体工商户和自然人主播测算。

4. 查看测算结果：
   顶部卡片展示三种身份的税负对比；“广告/推广费扣除口径对比”展示全额扣除和 15% 限额扣除两种结果。

5. 核查计算过程：
   点击每张结果卡片右上角“核查”，可查看当前输入代入后的计算公式。

## 规则更新

默认规则文件位于：

```text
rules/current-cn-tax-rules.json
```

如法律法规或政策口径调整，可以按原结构修改该 JSON 文件。页面也支持重新读取规则和导入同结构的规则 JSON。

如果部署到线上环境，可以把 `src/app.js` 中的 `RULES_URL` 改为远程规则文件地址，实现规则文件独立更新。

## 适用边界

当前版本主要用于年度税负粗算和身份选择对比，不覆盖以下情况：

- 核定征收
- 地方特殊优惠政策
- 社保、公积金和个别平台代扣差异
- 具体业务合同、发票品目、税目认定差异
- 复杂关联交易或企业重组安排
